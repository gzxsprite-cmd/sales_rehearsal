# 01 V0.1 Scope And Goal

## V0.1 goal

The first goal is to make a GitHub Pages demo that can be opened directly and used to demonstrate the core idea to business stakeholders.

The priority is:
1. Pages demo works
2. user flow is clear
3. visual and interaction feel right
4. business concept is understandable

## In scope for v0.1

- one product: VMS
- one main customer role: Project Director
- one main scenario: first introduction / value establishment
- one lightweight setup step
- one multi-turn chat rehearsal
- one structured review page
- one knowledge coaching section
- one retry / restart path

## Out of scope for v0.1

- real LLM integration
- backend
- database
- login / user management
- organization-level reporting
- multi-role meeting simulation
- production-grade architecture
- enterprise system integration
- universal product coverage

## V0.1 success criteria

The demo is successful if:
- a stakeholder can open it directly in browser
- the main flow is complete
- the user can understand the product in 2 minutes
- the simulated chat feels plausible
- the review page clearly shows value
- the knowledge coaching feels targeted

## Key product principle

V0.1 is not trying to prove technical sophistication.

V0.1 is trying to prove that:
- the concept makes sense
- the flow works
- the business side can see value
